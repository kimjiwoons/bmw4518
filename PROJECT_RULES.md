# 프로젝트 작업 규칙

## 필수: 세션 로그 기록

이 프로젝트는 session_log.md 파일로 모든 작업을 추적한다.
**모든 작업에서 아래 규칙을 반드시 따를 것.**

### 기록 시점

1. **작업 시작 시**
   - "현재 진행 중" 섹션에 작업 내용 기록

2. **파일 수정/생성 후**
   - 수정/생성 파일명 기록
   - 변경 내용 간단히 기록

3. **작업 완료 시**
   - "현재 진행 중"에서 삭제
   - "완료된 작업" 테이블에 추가
   - 결과 기록 (성공/실패/부분완료)

4. **에러/이슈 발생 시**
   - "발생한 이슈 및 해결" 테이블에 즉시 기록
   - 원인과 해결 방법 상세히 기록

5. **새로운 TODO 발견 시**
   - "다음 단계 (TODO)" 섹션에 추가

### 기록 형식

- 작업명: 무엇을 했는지 명확하게
- 파일: 수정하거나 생성한 파일 경로
- 내용: 구체적으로 뭘 변경했는지
- 결과: 성공/실패/진행중

### 주의사항

- 작업 내용이 많아도 **절대 로그 기록을 생략하지 말 것**
- 사소한 수정도 기록할 것
- 대화가 길어져도 이 규칙은 항상 유지
- 대화 종료 전 반드시 session_log.md 업데이트
